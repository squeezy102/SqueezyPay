# Database

SqueezyPay uses SQLite via SQLAlchemy. The database file lives next to the `backend/` directory (`squeezypay.db`) in dev mode, and in `%APPDATA%\SqueezyPay\squeezypay.db` in the packaged installer. It never leaves the machine. Schema changes are managed with Alembic.

## Migrations

Always use Alembic to change the schema. Never modify models without a corresponding migration.

```powershell
cd backend
.\venv\Scripts\Activate.ps1

# Apply all pending migrations (run on first install and after pulling changes)
alembic upgrade head

# Create a migration after changing models.py
alembic revision --autogenerate -m "short description"

# Roll back one migration
alembic downgrade -1

# Show current revision
alembic current

# Show migration history
alembic history
```

Review autogenerated migrations before committing. The `upgrade()` and `downgrade()` functions should be verified manually — autogenerate is not always complete or correct.

**Important:** autogenerate only captures the *diff* between the current model and the existing database. If the database was created via `Base.metadata.create_all()` before migrations existed, autogenerate will miss the `CREATE TABLE` statements entirely and only emit the delta. Always verify that the initial migration in a new chain creates all tables from scratch.

Migration integrity is verified by `backend/tests/test_migrations.py` (marked `@pytest.mark.slow`). Run it with `pytest -m slow` to verify the full chain against a fresh SQLite file.

Migration scripts are in `backend/alembic/versions/`.

## Backup

The entire database is a single file: `backend/squeezypay.db`. Copy it anywhere to back it up. Restoring is as simple as replacing the file (stop the backend first).

The database is gitignored. Never commit it.

## Resetting the database

To reset to a clean state:
```powershell
cd backend
del squeezypay.db
alembic upgrade head
```

This permanently deletes all data. The encryption key in your environment variables remains valid for future use.
